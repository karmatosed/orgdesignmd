## Overview

WordPress.org is the public face of the open source publishing platform — a multi-property site spanning marketing pages, news, plugin/theme directories, showcase, learning resources, and developer documentation. The design language is **Editorial Open Source**: confident serif headlines paired with clean sans-serif UI, a distinctive Blueberry accent, and campaign accent colors reserved for feature blocks.

The brand should feel **trustworthy, welcoming, and technically credible**. WordPress.org is a **federated visual brand** implemented across several stacks — not one uniform child-theme tree. Most properties use [wporg-parent-2021](https://github.com/WordPress/wporg-parent-2021) (FSE child themes or main-theme patterns); **Profiles** (BuddyPress legacy) and **Jobs** (standalone `jobswp`) are structural outliers.

This file is the **canonical [DESIGN.md](https://github.com/google-labs-code/design.md) artifact**: full YAML tokens, global prose, and cross-section component matrix. Property-specific typography, layout tables, component prose, and Do's live in **[stack guides](stacks/README.md)** — read `DESIGN.md` plus the relevant stack when building for one property.

## Layout

- **Content column:** 680px (`spacing.content-max`) — articles, about, prose.
- **Wide column:** 1160px (`spacing.wide-max`) — homepage, directories, showcase. News uses 1070px in its child `theme.json`. Photos and Showcase override to **1760px**. Events hub overrides to **1600px** (`spacing.events-wide-max`).
- **Documentation article layout:** 248px TOC sidebar + 40px gap via `.has-three-columns`; article body stays at 680px.
- **Edge spacing:** 80px desktop; fluid clamp on tablet/mobile.
- **Block gap:** 20px default.
- **List/quote indent:** 30px horizontal margin.

## Elevation & Depth

Tonal layers, not shadows:

- Page → white or `#F6F6F6`
- Cards → white, hover shifts to `surface-dim`
- Header → Charcoal 2 with 1px Charcoal 1 top border
- Footer → Charcoal 1 solid
- Showcase depth → 10px border at `rgba(30, 30, 30, 0.1)`, not box-shadow

Button states: primary → deep-blueberry (hover) → charcoal-1 (active).

**Architectural Minimalism** — the 2px button radius is a core brand marker.

| Badges | Pill | `rounded.full` |

## Do's and Don'ts

Global guardrails — section-specific rules are in [section guides](stacks/README.md).

- Do use YAML `badge-*` tokens for contribution medals — each WordPress team has a distinct color.
- Do keep button radius at 2px across all properties.
- Do use one accent family per homepage section.
- Do respect 680px / 1160px layout widths (unless a stack doc overrides wide size).
- Do use Charcoal 4 (`on-surface-variant`) for metadata only.
- Don't put full accent palettes in component tokens — pick one tint per section.
- Don't use gradients on buttons or navigation.
- Don't apply showcase 20px frames to directory UI outside Showcase.
- Don't use Anton or display-cta scale outside the main homepage and Enterprise cover.
- Don't simplify contribution medals to one color; each WordPress team has a distinct badge color.

Links in prose: Blueberry, underlined default, no underline on hover.
